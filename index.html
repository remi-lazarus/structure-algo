Structures Algorithmiques

Un algorithme est une suite d'instructions ordonnées permettant de résoudre un problème. Il peut être représenté sous forme d'algorigramme (graphique) ou de pseudo-code (littéral).

▶
Structure Séquentielle linéaire

Les instructions s'exécutent les unes après les autres, dans l'ordre où elles sont écrites. C'est la structure de base de tout algorithme.

DÉBUT
↓
Action 1
↓
Action 2
↓
Action 3
↓
FIN
Début Action_1 Action_2 Action_3 Fin
⇄
Structure Alternative si…alors

On évalue une condition booléenne. Selon le résultat (vrai ou faux), on exécute l'une ou l'autre des séquences — elles s'excluent mutuellement.

DÉBUT
↓
condition
?
VRAI
↓
Séquence A
FAUX
↓
Séquence B
↓
FIN
SI condition ALORS Séquence_A SINON Séquence_B FIN SI
// Variante sans SINON
SI condition ALORS Séquence_A ← si faux : on passe FIN SI
if (condition) { // Séquence A } else { // Séquence B }
if condition: # Séquence A else: # Séquence B

Structures Répétitives (Itératives) boucles

On répète un bloc d'instructions. Il existe trois formes selon qu'on connaît ou non le nombre d'itérations, et selon le moment du test de la condition.

① FAIRE … JUSQU'À

La séquence est exécutée au moins une fois. Le test de la condition a lieu après chaque itération.

DÉBUT
↓
Séquence
↓
cond
?
FAUX ↩ retour boucle
VRAI ↓
FIN
FAIRE Séquence JUSQU'À condition fausse
② TANT QUE … FAIRE

Le test est effectué avant chaque itération. Si la condition est fausse dès le départ, la séquence n'est jamais exécutée.

DÉBUT
↓
cond
?
FAUX → sortie
VRAI ↓
Séquence
↩
FIN
TANT QUE condition vraie FAIRE Séquence FIN TANT QUE
while (condition) { // Séquence }
while condition: # Séquence
③ POUR … FAIRE

On connaît à l'avance le nombre d'itérations. Un compteur i évolue de la valeur initiale jusqu'à la valeur finale.

DÉBUT
↓
i = 0
↓
i ≤ N
?
NON→sortie
OUI ↓
Séquence
i++ ↩
FIN
POUR i allant de 0 à N FAIRE Séquence FIN POUR
for (i=0; i<=N; i++) { // Séquence }
for i in range(0, N+1): # Séquence
⚠ Point de vigilance : Pour que toute boucle puisse se terminer, la condition doit obligatoirement finir par changer de valeur. Une boucle infinie bloque le programme !
