Workflow Git — étape par étape
1
Cloner le dépôt (une seule fois)
Récupère une copie locale du projet sur ton ordinateur.
$ git clone https://github.com/votre-classe/jeu-aventure
$ cd jeu-aventure
2
Créer ta branche de travail
Chaque élève travaille sur sa propre branche. Ne jamais modifier directement main !
# Remplace "intro" par foret / caverne / boss selon ton rôle
$ git checkout -b scene/intro
Switched to a new branch 'scene/intro'
3
Coder ta scène
Ouvre ton fichier dans VS Code et réalise les missions indiquées en commentaire (sections # 🎯 MISSION).
$ code scenes/intro.py
4
Sauvegarder avec un commit
Après chaque avancée significative, fais un commit. Un commit = une tâche précise.
$ git status # voir les fichiers modifiés
$ git add scenes/intro.py # préparer le fichier
$ git commit -m "feat: ajout de l'énigme scène intro"
5
Envoyer ta branche sur le serveur
Partage ton travail avec le groupe. À faire après chaque commit important.
$ git push origin scene/intro
Branch 'scene/intro' set up to track remote branch.
6
Fusionner dans main (en groupe)
Quand une scène est terminée, le groupe intègre la branche dans main.
$ git checkout main
$ git merge scene/intro
Merge made by the 'ort' strategy.
$ git push origin main
💬
Bien écrire ses messages de commit
ℹ️ Un bon message de commit explique ce qui a changé et pourquoi, pas comment. Il sera lu par tes coéquipiers et par toi dans 3 semaines !
feat: Nouvelle fonctionnalité
fix: Correction de bug
style: Texte narratif
docs: README
refactor: Réécriture
✅ feat: ajout combat contre le loup avec boucle TANT QUE
❌ modif  /  ça marche  /  update fichier
⚡
Résoudre un conflit de merge
⚠️ Un conflit se produit quand deux élèves ont modifié les mêmes lignes du même fichier. C'est normal — Git demande juste de choisir quelle version garder.
<<<<<<< HEAD (ta version)
print("Bonjour aventurier !")
=======
print("Bienvenue dans le jeu !")
>>>>>>> scene/intro (version du camarade)
1
Ouvrir le fichier conflictuel dans VS Code
Git indique les fichiers en conflit avec git status
2
Choisir (ou combiner) les deux versions
Supprimer les marqueurs <<<, ===, >>> et garder le bon code
3
Finaliser le merge
$ git add scenes/intro.py
$ git commit -m "fix: résolution conflit scène intro"

Scénario — La Quête du Code

Contexte narratif Détail des scènes Missions
📜
L'histoire
Dans un monde où les algorithmes font tourner la magie, un Sorcier du Code a semé le chaos en lançant des boucles infinies sur le royaume. Chaque route, chaque pont, chaque porte est bloqué dans une répétition sans fin.

Vous êtes un aventurier qui s'est réveillé seul dans une forêt. Votre mission : traverser les terres maudites, vaincre les créatures corrompues par le Sorcier, et mettre fin à sa boucle infinie une bonne fois pour toutes.

Seul un algorithme bien structuré peut vous sauver.
🗺️
Les 4 scènes du jeu
⚡ Scène 1 — Le Réveil Élève 1

Le joueur se réveille dans la forêt avec un vieux sac à ses pieds. Premier choix : fouiller ou ignorer le sac (potion de soin + 5 pièces d'or).

🎯 Missions : Ajouter une scène narrative, proposer un objet bonus, créer une énigme avec pénalité/bonus.

Structure : Séquentielle Alternative
🌲 Scène 2 — La Forêt Maudite Élève 2

Combat contre un Loup Géant (30 PV). Le joueur peut attaquer, utiliser une potion ou fuir. Le loup contre-attaque à chaque tour.

🎯 Missions : Ajouter une énigme post-combat, créer un marchand conditionnel (si assez d'or).

Structure : Alternative TANT QUE
🪨 Scène 3 — La Caverne Élève 3

Exploration de 3 salles (boucle POUR), piège à désamorcer en devinant un code (boucle TANT QUE, 3 essais), puis un Troll endormi à gérer.

🎯 Missions : Ajouter une salle de cristaux, améliorer le piège avec indices chaud/froid.

Structure : POUR TANT QUE Alternative
🧙 Scène 4 — Le Boss Final Élève 4

Combat contre le Sorcier du Code (80 PV). Bonus selon l'inventaire, coups critiques, salve de sorts tous les 4 tours, rage sous 50% PV.

🎯 Missions : Dialogue pré-combat avec bonus, invocation de squelettes, difficulté adaptative.

Structure : Séquentielle Alternative TANT QUE POUR

Aide-mémoire Git rapide
Commande Ce qu'elle fait Quand l'utiliser
git status Affiche les fichiers modifiés Avant tout commit
git add <fichier> Prépare un fichier pour le commit Après modification
git commit -m "msg" Sauvegarde un point dans l'historique Après chaque tâche
git push origin <branche> Envoie la branche sur le serveur Pour partager son travail
git pull Récupère les derniers changements En début de séance
git log --oneline Affiche l'historique des commits Pour naviguer l'historique
git branch Liste toutes les branches Pour vérifier où on est
git checkout <branche> Change de branche Pour naviguer entre branches
git merge <branche> Fusionne une branche dans la courante Pour intégrer le travail
git diff Montre les modifications en cours Avant de commiter
